FB_VibratorControl
Вибраторы 4шт : `stBunker.MotorVibrator[1-4]`

ФБ вызывается внутри FB_BunkerControl
## Входы ФБ
- Массив моторов Bunker.MotorVibrator
- Сигнал Enable (Работа блока возможна только при сигнале TRUE, работа немедленно останавливается при потере сигнала)
- Сигнал ручного запуска моторов 
- Частота работы мотора (Гц)
- Время активной работы вибратора (TIME)
- Время паузы вибратора (TIME)
- Время паузы ФБ (TIME)
- Количество срабатываний (INT)

## Выходы ФБ
- Моторы в работе
- Время до следующего срабатывания мотора (TIME)

## Алгоритм

Контакторы моторов включены, если есть сигнал ФБ Enable - TRUE.

При выключенном ФБ Enable, сигнал ручного запуска последовательно запускает контактор мотора, потом даёт сигнал ПУСК на частотный преобразователь (ЧРП). При включённом ФБ Enable и сигнале ручного запуска, даётся только ПУСК на ЧРП. При активном ручном запуске моторов, все таймеры внутри ФБ останавливаются.

У ФБ есть подход. Внутри подхода действует алгоритм: моторы срабатывают столько раз, сколько указано в уставке, соблюдая паузу. Мотор работает определенное время (время активной работы вибратора), потом соблюдает паузу (время паузы вибратора), если ещё не дошёл до максимума количества срабатываний, повторяет свой цикл. Когда мы приходим к максимуму количества срабатываний, мы соблюдаем паузу (время паузы фб), сбрасываем счётчик срабатывания и начинаем цикл заново.
